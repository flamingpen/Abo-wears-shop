The Netlify deploy errored, with the following guidance provided:

Diagnosis

- The TypeScript typecheck fails with a top-level await error in artifacts/mockup-sandbox/vite.config.ts. See the error in the logs at [line 68](#L68) which references vite.config.ts(22,13). The package run that failed is artifacts/mockup-sandbox (see [line 70](#L70)).
- The error is TS1308: "'await' expressions are only allowed within async functions and at the top levels of modules." That means the TypeScript compiler settings for this workspace/package do not allow top-level await (module/target settings), or the await is used outside an async function.

You can inspect the exact source location here:
- vite.config.ts (problem line ~22): https://github.com/flamingpen/Abo-wears-shop/blob/main/artifacts/mockup-sandbox/vite.config.ts#L22
- package tsconfig: https://github.com/flamingpen/Abo-wears-shop/blob/main/artifacts/mockup-sandbox/tsconfig.json

Solution

Choose one of the two fixes below.

Option A — Enable top-level await in TypeScript (recommended if you intentionally use top-level await)
1. Verify which tsconfig is used for the mockup-sandbox package (artifacts/mockup-sandbox/tsconfig.json or the root tsconfig it extends).
2. Edit that tsconfig's compilerOptions to allow top-level await by using an ES module module setting and an appropriate target. For example, update artifacts/mockup-sandbox/tsconfig.json to include:

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "Node",
    ...
  }
}
```

- Commit the change and re-run the build. This tells TypeScript to compile as an ES module where top-level await is permitted.

Option B — Remove top-level await by wrapping code in an async function (no tsconfig changes)
1. Open artifacts/mockup-sandbox/vite.config.ts and replace the top-level await with an async config factory. For example, if the file currently does something like:

```ts
const env = await loadEnv(...);
export default defineConfig({ /* uses env */ });
```

Change it to:

```ts
export default defineConfig(async () => {
  const env = await loadEnv(...);
  return { /* uses env */ };
});
```

2. Save, commit, and re-run the build.

Notes
- Make sure you edit the tsconfig that TypeScript actually uses for artifacts/mockup-sandbox (see the package tsconfig linked above). If the package extends the root tsconfig, adjust the proper file.
- After applying either fix, re-run pnpm run build on your CI (Netlify) to confirm typecheck passes.
